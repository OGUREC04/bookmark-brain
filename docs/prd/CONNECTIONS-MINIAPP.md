# Бриф для фронта: Связи между заметками (Mini App)

## Контекст за 30 секунд

Бэкенд-слой связей **готов, смёржен в `main` и проверен против реального pgvector** (миграция + весь SQL отработали вживую). Связи между заметками строятся автоматически по смыслу; API отдаёт «похожие», граф и семантический поиск. **Осталась видимая часть — этот фронт.**

- Бэкенд-эпик: `bookmark-brain/docs/epics/connections-mvp.md`
- Поведение (БТ): `bookmark-brain/docs/requirements/бт-11-связи.md`
- Решения: реальный текст в эмбеддинге (не только ИИ-выжимка); граф on-demand (локальный сам, полный по кнопке с кэшем); тяжёлую раскладку считает **клиент**, сервер хранит координаты.

## Что строим (3 куска)

1. **Секция «Связано»** в карточке заметки — топ-5 похожих + «Посмотреть все».
2. **Таб «Граф»** — локальный (эго) граф вокруг заметки на лету + полный граф по кнопке с кэшем.
3. **Семантический режим в поиске** — переключатель «по смыслу / обычный».

## Контракты API (проверены вживую, не поменяются)

База: `/api/v1`, авторизация — JWT из Telegram initData (как у существующих вызовов в `lib/api.ts`).

| Метод | Ответ |
|---|---|
| `GET /bookmarks/{id}/related?limit=5&all=false` | `{ items: [{id,title,summary,item_type,weight,created_at}], total }` (`total` — истинное число связей для бейджа; `all=true` — все) |
| `GET /graph/local?center={id}&depth=2` | `{ nodes:[{id,title,item_type}], edges:[{from,to,weight}], center }` — эго-граф на лету |
| `GET /graph` | `{ nodes, edges, layout, stale, node_count, built_at }` — полный граф; `layout` = кэш координат `[{id,x,y}]` или `null`; `stale=true` → пора пересобрать |
| `POST /graph/build` body `{ nodes:[{id,x,y}] }` | `{ node_count, saved }` — сохранить раскладку, посчитанную клиентом (≤350 узлов) |
| `POST /search/` body `+ mode:"hybrid"\|"semantic"` | как сейчас (`{results, total, query, summary}`), плюс поле `mode` в запросе |

## Библиотеки графа (рекомендация)

- **Локальный граф** (мало узлов, эго): **`react-force-graph-2d`** — canvas, лёгкий для мобильного Telegram WebView. Считает раскладку сам, мгновенно. Рёбра либы — `links:[{source,target}]`.
- **Полный граф** (много узлов): при росте — **`@cosmograph/react`** (WebGL, 100k+ узлов). Для MVP можно тем же `react-force-graph-2d`, а Cosmograph включить, когда узлов реально много.
- **Раскладку полного графа считает клиент** (force-graph отдаёт координаты узлов `x/y` после симуляции) → шлём их `POST /graph/build` → дальше открытие из кэша мгновенно.

## UX-флоу

**Секция «Связано» (DetailScreen):**
1. Открыл заметку → `getRelated(id, 5)`.
2. `total === 0` → секцию не показываем.
3. `total > 0` → заголовок «🔗 Связано (N)» + топ-5 карточек (тап → переход на связанную заметку). Если `total > 5` → кнопка «Посмотреть все» → `getRelated(id, all=true)`.

**Таб «Граф»:**
1. **Локальный** (по умолчанию при заходе с заметки): `getGraphLocal(centerId, 2)` → рисуем force-graph, центр подсвечен, тап по узлу → заметка. Мгновенно, без кэша.
2. **Полный**: кнопка «Построить граф» → `getGraph()`. Если `layout` есть и `stale=false` → рисуем по кэшированным координатам сразу. Если `layout=null` или `stale=true` → клиент считает раскладку (force-graph), показываем прогресс, по готовности `buildGraph(coords)` сохраняем.
3. Баннер «Граф устарел — обновить?» при `stale=true`.

## Ограничения репо (важно — из разведки)

- **НЕТ react-router** — навигация только через state-machine в `App.tsx` (TG iOS WebView ломает history API). Новый экран = новая ветка в switch, не `<Route>`.
- **Адаптер-паттерн**: `Bookmark`/graph-данные не используем в UI напрямую — трансформируем в `adapters.ts`, в компоненты идут структурные пропсы.
- **DS-компоненты залочены 1:1** к дизайн-системе (`components/ds/`) — не рефакторим в классы, CSS-переменные из tokens.css.
- **Мутации** — через `runAction(async () => {...})` (единый error-handling + хаптика). Хаптика: `hapticImpact`/`hapticNotify`.
- Типы в `lib/api.ts` правим в том же PR, что и контракты бэка (без рассинхрона).
