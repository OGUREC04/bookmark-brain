# PRD — Извлечение контекста ссылки (z9q)

**Статус:** черновик для бэкенд-сессии
**Бед:** `bookmark-brain-z9q` (P1)
**Создан:** 2026-05-31 (по итогу E2E-прогона Mini App)

## Проблема

Когда юзер сохраняет **голую ссылку** (только URL, без текста), Mini App показывает заголовок = хост (`linkedin.com`) и плейсхолдер «Brain читает ссылку…» / «контекст не извлёкся — открой источник». Но дальше ничего не происходит: `summary`, `key_ideas`, `title` остаются пустыми, потому что бэкенд **не умеет вытаскивать контент по URL**. Юзер видит просто `linkedin.com` и кнопку «открыть источник» — непонятно что за ссылка, зачем сохранял.

Скрин из прода: заголовок `linkedin.com`, тело — «Brain читает ссылку…», и так навсегда (AI-пайплайн для ссылок не реализован).

## Решение (предлагаемое)

Добавить в AI-воркер шаг **fetch + extract** для закладок типа ссылка:

1. **Детект ссылки** — `content_type == "link"` или `raw_text` это URL без доп. текста.
2. **Fetch HTML** по URL (httpx, таймаут ~10с, user-agent как у браузера, follow redirects).
3. **Извлечь метаданные** в порядке приоритета:
   - **OpenGraph** (`og:title`, `og:description`, `og:site_name`, `og:image`)
   - **Twitter Cards** (`twitter:title`, `twitter:description`)
   - **Стандартные** (`<title>`, `<meta name="description">`)
   - **JSON-LD** (`application/ld+json` — Article, особенно для новостей/блогов)
4. **Заполнить** `title` (из og:title), `summary` (из og:description), при наличии — прогнать через LLM для `key_ideas`.
5. **ai_status** → `completed`. Mini App уже рендерит `summary`/`key_ideas` когда они есть (фронт готов).

## Корнер-кейсы (обязательно проработать)

1. **Auth-walled (LinkedIn, X/Twitter, Instagram, FB)** — отдают login-страницу или 999/403. OG-теги часто всё равно присутствуют в `<head>` даже за логином — попробовать вытащить. Если пусто → graceful: `summary=null`, плейсхолдер «контекст не извлёкся — открой источник» (фронт уже показывает).
2. **Пейволл (NYT, Medium)** — OG-теги обычно есть (для шеринга), их хватает.
3. **Редиректы / шортлинки** (t.co, bit.ly) — follow до финального URL, метаданные с него.
4. **Не-HTML контент** (PDF, картинка, видео по прямой ссылке) — по `Content-Type`. PDF → пересечение с Phase 3B (documents). Картинка → og недоступен, показать тип.
5. **Таймаут / медленный сайт** — жёсткий лимит, не блокировать воркер. Фейл → retry по существующему cron (как embedding retry).
6. **Мусорный og:description** (cookie-баннер, «Включите JS») — эвристика: если description содержит «enable JavaScript» / «cookies» и короче N символов → отбросить, оставить хост.
7. **Кириллица / кодировки** — уважать `<meta charset>` и `Content-Type charset`, не ломать UTF-8.
8. **SSRF-защита** — НЕ фетчить `localhost`, `127.0.0.1`, приватные IP-диапазоны (10/8, 172.16/12, 192.168/16), `file://`, `gopher://`. Только http/https на публичные адреса. **Критично** — это серверный fetch по юзер-вводу.
9. **Размер ответа** — лимит на скачивание (~2-5 МБ), не тянуть гигабайтные страницы; читать только `<head>` если возможно (стримить до `</head>`).
10. **Дубли** — если по этому URL уже есть закладка с извлечённым контекстом, можно переиспользовать (опционально, не для MVP).

## Контракт (фронт уже готов)

Поля в `Bookmark` (см. `bookmark-brain-miniapp/src/lib/api.ts`), которые фронт рендерит когда не пустые:
- `title: string | null` → заголовок заметки (вместо хоста)
- `summary: string | null` → блок под заголовком
- `key_ideas: string[] | null` → блок «brain» (через `·`)
- `ai_status: "completed"` → убирает плейсхолдер «Brain читает ссылку…»

**Менять схему `Bookmark` не нужно** — поля уже есть. Только заполнить их в воркере.

## Связь с существующим

- AI-воркер: `backend/app/worker.py` (или его пакет после сплита 0dj) — добавить шаг в пайплайн `classify → embed → tag`.
- Уважать `AI_PROVIDER` абстракцию для LLM-обогащения `key_ideas`.
- SSRF + fetch-утилита — новый модуль `backend/app/services/link_fetch.py` (или аналог).

## Out of scope (не сейчас)

- Полный article-extraction (readability/trafilatura) — MVP только OG/meta.
- Рендер JS-страниц через headless-браузер (Playwright) — дорого, отдельный трек если OG не хватит.
- Скриншоты/превью-картинки.

## Success criteria

- Сохранил публичную ссылку (habr, arxiv, youtube, обычный блог) → через N секунд в Mini App виден человеческий заголовок + краткое описание.
- Auth-walled ссылка (LinkedIn) → либо OG-данные, либо аккуратный плейсхолдер (не вечное «читает…»).
- Ноль SSRF-векторов (приватные адреса не фетчатся) — есть тест.
