# Бриф для бэка: загрузка медиа (голос + файлы) из Mini App

**Статус:** бриф (frontend-driven), 2026-06-10
**Тикет:** bookmark-brain-ti0
**Связан:** voice-features (бот STT), documents (бот extract)
**Кто инициатор:** Mini App (юзер хочет записать голос / прикрепить файл прямо в приложении)

> Описывает, что нужно фронту, чтобы заработали кнопки 🎤/📎 в QuickCreate.
> Здесь **продуктовая развилка A/B** — её решает юзер. Технические под-решения — за бэком.

## Проблема

В `QuickCreateSheet` кнопки **🎤 голос** и **📎 файл** намеренно `disabled` (подсказка
«вложения — в боте»). Причина — на бэке **нет приёма медиа из Mini App**:

- **STT** (`bot/services/stt.py`, провайдеры openai/groq/**yandex** SpeechKit; для длинных
  голосовых — через Yandex S3) живёт в **боте**.
- **Doc-extract** (`bot/services/extractor.py → extract_text`, PDF/DOCX) живёт в **боте**.
- **Backend FastAPI этого кода не имеет** (отдельный пакет/процесс). Закладку с готовым
  текстом создаёт бот: скачал из Telegram → STT/extract → `POST /bookmarks/` с `raw_text`.

Mini App в Telegram-сообщения слать не может — нужен HTTP-путь через backend.

## 🔱 РАЗВИЛКА (решает юзер)

| | **A. Полная реализация** | **B. Дешёвая заглушка** |
|---|---|---|
| Суть | Backend-эндпоинт приёма медиа + STT/extract; фронт пишет звук (MediaRecorder) и шлёт файл | Кнопки → deep-link `tg://resolve?domain=<bot>` — открывают чат с ботом, где медиа уже работает |

нужно «прямо сейчас» → **B** как промежуток, A потом. Ниже — контракт для **A**.

## Path A — что нужно фронту от бэка

### 1. Эндпоинт приёма
`POST /api/v1/bookmarks/upload` (multipart/form-data), JWT как у всех Mini App вызовов.
- `file`: бинарь (аудио или документ)
- `kind`: `"voice" | "document"` (или определять по MIME)
- опц. `caption`: текст-приписка (как в боте — добавляется в начало `raw_text`)

### 2. Асинхронность через `ai_status` (как в 0rn)
STT/extract — медленные. Фронт **не должен** ждать синхронно. Контракт:
- эндпоинт быстро создаёт закладку-«заготовку» (`ai_status="pending"` или новый `"transcribing"`),
  кладёт `media_file_id`/storage-ключ, **возвращает `BookmarkResponse` сразу**;
- воркер делает STT/extract → пишет `raw_text` (+ `transcription` для голоса) → дальше обычный
  pipeline (classify/embed) → `ai_status="completed"`;
- фронт рендерит «Brain слушает…/читает…» по `ai_status` и **рефетчит** (поллинг уже есть в
  Mini App по `ai_status`).
- При фейле STT/extract → `ai_status="failed"` + `ai_error` (фронт покажет ошибку).

### 3. Форматы и лимиты (фронту нужно знать заранее)
- **Аудио:** браузерный MediaRecorder даёт **WebM/Opus** (Chrome/Android) или **mp4/AAC** (iOS
  Safari/Telegram WebView). Yandex SpeechKit принимает OggOpus/LPCM/MP3 → **вероятно нужен
  транскод** на бэке. Фронту надо: какой MIME слать, есть ли транскод, какой максимум секунд.
- **Документы:** те же типы, что бот (PDF/DOCX). Лимит размера — бот рубит **>20 МБ**; для
  Mini App бэк должен озвучить свой лимит (фронт покажет/зарубит до отправки).
- Ответ при превышении лимита/неподдерж. формате — **422** с понятным detail (фронт покажет тост).

### Под-решение бэка (НЕ блокирует фронт)
Где взять STT/extract в backend-контуре:
- вынести `stt.py`/`extractor.py` в **общий пакет** (импортит и бот, и воркер), воркер зовёт; **или**
- эндпоинт кладёт файл в S3/temp → enqueue воркер-джоб (воркер уже рядом с этим кодом).
Фронту всё равно как — важен контракт `ai_status` + ответ эндпоинта.

## Edge-кейсы (важны фронту)
| # | Кейс | Ожидание фронта |
|---|------|-----------------|
| 1 | iOS Telegram WebView не даёт MediaRecorder/мик | фронт деградирует на B-ссылку «записать в боте»; бэк тут ни при чём, но нужно знать заранее |
| 2 | Длинное голосовое (>30с / >1МБ для Yandex sync) | бэк сам решает (S3-путь как в боте); фронт просто шлёт файл |
| 3 | STT вернул пустоту/мусор | `ai_status` терминальный, `raw_text` пустой → фронт покажет «не распозналось», даст переотправить/править текстом |
| 4 | Аплоад прервался (сеть) | эндпоинт не создаёт заготовку без файла; фронт ретраит |
| 5 | Файл не того типа | 422 до создания закладки |
| 6 | Прогресс загрузки | фронт сам по XHR/fetch; бэку слать ничего не надо |

## Открытые вопросы (решает бэк)
1. Где живут STT/extract для backend-контура (общий пакет vs воркер).
2. Транскод аудио (WebM/mp4 → OggOpus) — на бэке или требовать конкретный MIME от фронта.
3. Лимит размера и длины для Mini App (может отличаться от Telegram 20 МБ).
4. Sync-заготовка vs полностью async upload (стриминг больших файлов).
5. Хранение исходника (S3 как у голосовых в боте? lifecycle? — ср. тикет d0v).

## Out of scope / фазирование
- **Фаза 1 — только голос** (главный запрос юзера). Файлы (PDF/DOCX) — фаза 2 тем же эндпоинтом.
- Видео-кружки, фото-OCR — отдельно (как в боте Phase 3B-C).

`QuickCreateSheet`: снять `disabled` с 🎤/📎; MediaRecorder + `<input type=file>`; состояние
«Brain слушает…» по `ai_status` (поллинг уже есть); тосты ошибок по 422/`ai_error`.
Типы в `src/lib/api.ts` — добавить `uploadMedia(file, kind, caption?)`.
