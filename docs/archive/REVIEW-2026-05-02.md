# Code & Architecture Review — 2026-05-02

## Scope
Full review of backend (FastAPI + SQLAlchemy + arq worker) and bot (aiogram 3.x + Redis state).
Three parallel reviews: backend code, bot code, architecture.

## CRITICAL (fix immediately)

### C1. SECRET_KEY has forgeable default
- **File:** `backend/app/config.py:31`
- **Issue:** `SECRET_KEY: str = "change-me-in-production"` — if .env missing, anyone can forge JWT
- **Fix:** Remove default, make field required. Add startup guard.

### C2. BOT_SECRET defaults to empty string
- **Files:** `bot/config.py:12`, `backend/app/config.py`
- **Issue:** Empty string = `compare_digest("", "")` returns True = auth bypassed
- **Fix:** Remove default, make required. Add `if not settings.BOT_SECRET` guard.

### C3. TLS verification disabled for GigaChat
- **Files:** `backend/app/services/ai_classifier.py:140`, `backend/app/services/embeddings.py:48`
- **Issue:** `httpx.AsyncClient(verify=False)` — MITM possible
- **Fix:** Bundle Sber CA certificate, use `verify="certs/sber_ca.pem"`

### C4. _pending_saves keyed by tg_id — saves wrong message
- **File:** `bot/handlers/start.py:20, 422-433`
- **Issue:** Dict keyed by user_id, but callback_data contains message_id. If user sends two messages, clicking "Yes" on first saves the second.
- **Fix:** Key dict on message_id, lookup from callback_data. Add 5-min expiry.

### C5. InaccessibleMessage not handled in callbacks
- **Files:** All callback handlers in bot
- **Issue:** `callback.message` can be `InaccessibleMessage` (>48h old) — `.edit_text()` raises AttributeError
- **Fix:** Guard with `if not isinstance(callback.message, Message): return`

## HIGH (fix before next release)

### H1. IDOR — bookmark fetch without user_id in WHERE
- **File:** `backend/app/api/bookmarks.py:142-155`
- **Fix:** Add `Bookmark.user_id == current_user.id` to WHERE, return 404 (not 403).

### H2. N+1 queries in tag creation
- **File:** `backend/app/services/bookmark_processor.py:198-227`
- **Fix:** Use `INSERT ... ON CONFLICT DO NOTHING RETURNING id` + batch insert.

### H3. bookmarks_count inflated on reprocess
- **File:** `backend/app/services/bookmark_processor.py:189-193`
- **Fix:** Only increment on first creation, not on reprocess.

### H4. Worker double-commit race on task_list
- **File:** `backend/app/worker.py:153, 179`
- **Fix:** Wrap post-commit block in try/except with dedicated commit.

### H5. Worker directly calls Telegram API (coupling)
- **File:** `backend/app/worker.py:15-86`
- **Fix:** Extract Telegram notification into shared service or use Redis pub/sub.

### H6. Race in _rerender_at_bottom (concurrent taps)
- **File:** `bot/handlers/tasks.py:191-231`
- **Fix:** Per-(chat_id, bookmark_id) asyncio.Lock.

### H7. StateStore._get() lazy init race
- **File:** `bot/state_store.py:31-34`
- **Fix:** Eager init in __init__ or asyncio.Lock double-check.

### H8. bump_last_seen non-atomic read-modify-write
- **File:** `bot/state_store.py:98-107`
- **Fix:** Use Redis `SET key value EX ttl GT` (Redis 7.0+).

### H9. Service message deletion by offset heuristic
- **File:** `bot/handlers/tasks.py:210-217`
- **Fix:** Remove offset loop. Rely on `on_pin_service_message` handler.

### H10. httpx client without retry/limits
- **File:** `bot/api_client.py:12-16`
- **Fix:** Add `transport=httpx.AsyncHTTPTransport(retries=2)`, reduce timeout to 10s.

### H11. Token cache without TTL — stale JWT forever
- **File:** `bot/handlers/start.py:17`
- **Fix:** Store `(token, expires_at)` tuples, force re-auth on 401.

### H12. SQL via f-string, LIKE wildcards unescaped
- **File:** `backend/app/services/search.py:88-134`
- **Fix:** Escape LIKE wildcards. Consider SQLAlchemy Core expressions.

### H13. No rate limiting on API
- **File:** `backend/main.py`
- **Fix:** Add slowapi or custom middleware. Cap reprocess-all.

## Architecture Recommendations

- **Bookmark God Object** (30+ cols) → extract BookmarkAnalysis table
- **/health doesn't check deps** → add Postgres + Redis ping
- **API versioning** → `/api/v1/` before Mini App/iOS
- **Cursor-based pagination** → replace offset-based
- **Worker httpx per-job** → pool connections at worker level
- **CORS misconfiguration** → explicit origins, not `["*"]` with credentials
