# Команды бота

> Проверено: 2026-05-15 · класс: evergreen · конвенция обновления: `docs/README.md`

Все команды работают и в `@N0teeBot` (прод), и в `@bookmarkbrain_dev_bot` (дев).

## Основные

### `/start`
Первый запуск. Бот регистрирует тебя (привязывает Telegram ID к юзеру в базе) и показывает короткую подсказку — что можно делать.

### `/help`
Полная справка по командам — в самом боте, чтобы не ходить сюда.

### `/list`
Показывает последние закладки списком. Самые свежие сверху.

### `/search <запрос>`
Гибридный поиск: по смыслу + по точному вхождению одновременно. Возвращает топ результатов с саммари и ссылкой.

Пример: `/search про дизайн интерфейсов` найдёт и заметку «UX-патерны мобилок», и сохранённую статью где слово «дизайн» только в середине текста.

Без запроса (`/search` без аргумента) — бот напомнит как пользоваться.

### `/random`
Случайная закладка из всех твоих. Удобно для повторного просмотра старого.

### `/stats`
Сколько у тебя закладок всего, сколько по типам (текст / голос / документ), сколько ещё в обработке.

## Управление

### `/silent` (или `/silent on` / `/silent off`)
Тогглит тихий режим. В нём бот общается только реакциями (👀/👍/👎) и эфемерными подсказками. Чат остаётся как Saved Messages — твои сообщения, без болтовни бота.

`/silent` без аргумента — переключает текущее состояние. `/silent on` / `/silent off` — задаёт явно.

### `/clean`
Удаляет сообщения бота из чата. Защищает закреплённые task lists — их не трогает.

`/clean all` — удалит в том числе task lists. Использовать осторожно, отмены нет.

### `/reprocess`
Перезапускает AI-обработку для последней неудачной закладки (если `ai_status=failed`).

`/reprocess all` — принудительно переобрабатывает все закладки юзера. Долгая операция, использовать редко (например после смены провайдера эмбеддингов).

### `/todo`
Создаёт пустой task list, чтобы потом дополнять. Обычно task list рождается автоматически из обычного сообщения с несколькими строками-задачами — но можно начать вручную.

## Напоминания (Phase 2.5)

Три способа создать напоминание — выбирай по контексту.

### `/remind <текст> <когда>`
Явная команда. Без AI и закладки — сразу reminder.

Примеры:
- `/remind купить хлеб завтра в 9`
- `/remind позвонить маме через час`
- `/remind заплатить за квартиру 15.05`

Без времени (`/remind купить хлеб`) — бот спросит когда, ответь reply'ем.

### Strong intent (без команды)
Сообщение начинается с **«надо / нужно / не забыть / срочно / обязательно»** → бот покажет 3 кнопки 🔔 / 📝 / ✕ **до** AI-обработки:
- 🔔 — создать напоминание (если время есть в тексте — сразу, иначе спросит)
- 📝 — обычная закладка (AI-обработка пойдёт как всегда)
- ✕ — отмена

### Weak intent (через AI)
Обычное сообщение которое AI распознал как требующее напоминания → после save появится кнопка «🔔 Создать напоминание?». Менее агрессивно — не предлагает reminder на каждый текст.

### «Сделай напоминание» / «напомни» (Phase 2.6 T8)
Обычное сообщение, начинающееся с **«сделай напоминание …»** / **«напомни …»** / **«поставь reminder …»** / **«создай напоминание …»** — НЕ становится закладкой. Бот сразу делает reminder (если время есть — сразу, иначе спросит).

Примеры:
- `сделай напоминание купить молоко завтра в 9`
- `напомни в пятницу про отчёт`
- `напомни мне через час`

Reply на task_list с теми же триггерами тоже работает: «напомни в пятницу в 18» в reply на закреплённый список — создаст composite reminder на этот список.

### Длинные сообщения с задачами и датами (Phase 2.6 dispatcher)
Сообщения вида «завтра контрольная в 9, в пятницу зачёт в 18, ещё тесты Eltex» обрабатываются автоматически:
- **2+ дат** → бот создаёт по reminder'у на каждую задачу с датой («task_list_with_reminders»). Сообщает «✅ Создал 2 напоминания».
- **1 дата + несколько пунктов** → 3 кнопки «📋 Список / 🔔 Напоминание / ✕»: список с per-item reminder ИЛИ один composite reminder на весь текст ИЛИ отмена.
- **1 дата + один пункт** → один reminder автоматически.
- **дата без часа** («контрольная в пятницу») → бот спрашивает время через reply.

Если кликнуть «📋 Список» → бот создаст reminder на dated-пункт и оставит остальное чекбоксами; если «🔔 Напоминание» → один reminder с текстом всего сообщения.

### NL-edit task_list с реминдерами (Phase 2.6 cascade)
Reply на закреплённый task_list:
- **«удали 2»** — если у этого пункта был reminder, бот его отменит (молча, без отдельного сообщения)
- **«добавь врач 2026-05-20»** — создаст новый reminder автоматически (по дефолту 9:00 в твоей таймзоне)
- **«перенеси контрольную на пятницу»** — переставит fire_at у связанного reminder'а

В reply бот пишет короткое «✅ +1 напоминание» / «✅ перенёс 1 напоминание» / «✅ отменил 1 напоминание».

### `/reminders`
Список активных напоминаний (нумерованный). Reply на это сообщение:
- `отмени 1` / `удали 2` — отменить
- `перенеси 2 на завтра в 9` — изменить время
- `история` — показать выполненные за последние 30 дней

### Управление пришедшим напоминанием
Когда reminder сработал — приходит сообщение с кнопками:
- **✅ Выполнено** — отметить и закрыть
- **💤 Продлить** — указать новое время через reply

### Форматы времени (для всех команд)
`сегодня` / `завтра` / `послезавтра` / `в субботу` / `15 мая` / `15.05` / `через час` / `через 30 минут` / `утром` / `вечером` / `в 9` / `в 18:30` / комбинации.

## Не команды, но важно

### Просто текст
Любое не-команда сообщение — становится закладкой. Бот сам решает что это: обычная заметка, ссылка, чек-лист.

### Forward (пересланное)
Точно так же сохраняется, плюс бот видит источник пересылки.

### Голосовое / video note / audio
Распознаётся (Yandex SpeechKit), потом intent detection: задача / поиск / заметка. Дальше как обычная закладка с авто-тегом `#voice`.

### Документ (PDF / DOCX / TXT / MD)
Извлекается текст, прогоняется через AI-пайплайн. Сохраняется количество страниц.

### Reply на task list
Любой ответ на сообщение-чеклист = редактирование списка обычным языком: «сделал первое», «убери второе», «перенеси на завтра», «добавь купить молоко».
