# docs/ — карта документации

Этот файл — индекс и **конвенция обновления** `docs/`. Структура держится
не на неписаном знании: ниже зафиксировано что где лежит, к какому классу
относится и когда обновляется.

## Два класса документов

- **Evergreen-референс** — отражает *текущее* состояние, обновляется
  постоянно, живёт в корне `docs/`. Имеет в шапке строку
  `Проверено: YYYY-MM-DD` — дата последней сверки содержимого с кодом.
- **Снимок во времени** — point-in-time артефакт, устаревает намеренно,
  имеет дату в имени. После закрытия фазы переезжает в `docs/archive/`.

Правило: **в корне `docs/` — только evergreen + живой ROADMAP.** Любой
датированный снимок после фазы → `docs/archive/`.

## Корень docs/

| Файл | Назначение | Класс | Когда обновлять |
|---|---|---|---|
| `README.md` | Этот индекс + конвенция | evergreen | при изменении структуры docs/ |
| `ARCHITECTURE.md` | Архитектура, data model, auth, env, слои/контракты | evergreen | после деплоя фазы / смены архитектуры |
| `SPEC.md` | Что и зачем строим (продуктовая спека) | evergreen | при смене продуктового скоупа |
| `API.md` | Человекочитаемый обзор `/api/v1/` | evergreen | при изменении эндпоинтов |
| `BOT-COMMANDS.md` | Справочник команд бота (полный) | evergreen | при добавлении/изменении команд |
| `BOT-UX.md` | UX-принципы бота (clean chat, тон, паттерны) | evergreen | при смене UX-логики |
| `DEVELOPMENT-GUIDE.md` | Как запускать/деплоить/тестировать | evergreen | при смене dev/deploy-процесса |
| `TROUBLESHOOTING.md` | Известные грабли и решения | evergreen | при появлении новой грабли |
| `ROADMAP-2026-05.md` | Живой роадмап (датирован по месяцу) | living | по ходу планирования фаз |

> `BOT-COMMANDS.md` vs `BOT-UX.md`: первый — **что** за команды (справочник),
> второй — **как** бот общается (принципы). Не дублировать; ссылаться крест-накрест.

## Подпапки — жизненный цикл CCPM/ADR

| Папка | Назначение | Конвенция |
|---|---|---|
| `prd/` | PRD на фазу (≥1ч фича — обязателен, см. `~/.claude/rules/common/ccpm-boundaries.md`) | `docs/prd/<NAME>.md`; проблема+решение+edge cases+success |
| `epics/` | Декомпозиция эпика: линк на PRD + Beads root | `docs/epics/<name>.md` |
| `decisions/` | ADR — почему приняли техрешение | `NNNN-slug.md`, шапка Статус/Дата/Связано |
| `bugs/` | Датированные post-mortem | `YYYY-MM-DD-<slug>.md` |
| `archive/` | Устаревшие снимки во времени (session-context, review) | переносить сюда после фазы, не править |

## Кто принуждает обновление

1. **`~/.claude/rules/common/docs-with-code.md`** — обновлять доки в том же
   ответе, что и код.
2. **`~/.claude/hooks/check-docs-update.py`** (PreToolUse Edit|Write) —
   инжектит напоминание при правке скриптов в `/app/ /services/ /src/
   /scripts/ /lib/ /bot/`. Текст напоминания ссылается на эту конвенцию.
3. **`~/.claude/rules/common/ccpm-boundaries.md`** — обязательный PRD в
   `docs/prd/` + ADR рядом перед фазой ≥1ч.
4. **`.claude/rules/development-workflow.md`** — после деплоя фазы сверить
   evergreen-доки и обновить `Проверено:`.
