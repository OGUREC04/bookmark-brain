# ADR 0001 — GigaChat как основной LLM-провайдер для классификации

**Статус:** принято
**Дата:** 2026-04
**Связано с:** `backend/app/services/ai_classifier.py`

## Context

Нужен LLM для классификации заметок, тегирования и саммари. Требования:

- Доступен из РФ без VPN (юзеры — русскоязычные).
- Бесплатный или дешёвый стартовый тариф (продукт ещё без выручки).
- Достаточно качественный русский (категории + теги в одной модели).
- API совместимый с обычным HTTP (без проприетарных SDK с подводными камнями).

OpenAI и Anthropic заблокированы из РФ напрямую. DeepSeek — кандидат, но нужна валидация доступности и стабильности JSON.

## Decision

Использовать **GigaChat (Sber)** как primary `AI_PROVIDER` для классификации, тегов и саммари. Claude Sonnet оставляем как опциональный fallback (через `AI_PROVIDER=claude`) для случаев когда GigaChat не справляется.

GigaChat вызывается через `httpx` напрямую с OAuth-токеном — официальный python-SDK сырой и падает на нашем pipeline.

## Consequences

**Плюсы:**
- Бесплатный тариф покрывает MVP.
- Русский язык — нативный.
- Доступен из РФ.

**Минусы:**
- JSON-стабильность хуже чем у GPT-4 / Claude — иногда возвращает невалидный JSON, нужна defensive-парсилка.
- Embeddings 402 на бесплатном тарифе (см. ADR 0005 — почему Voyage).
- Лимиты бесплатного тарифа могут стать узким горлом при росте — нужен план Phase 4.5 (миграция на DeepSeek).
- Зависимость от российского провайдера — для не-РФ пользователей понадобится fallback (Claude уже подключён).
