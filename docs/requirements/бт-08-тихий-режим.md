# БТ-8: Тихий режим (silent mode)

> Бизнес-требования · living-doc «как ведёт себя сейчас»
> Обновлено: 2026-05-24
> Связано: PRD [`SILENT-MODE`](../prd/SILENT-MODE.md) · код `bot/handlers/settings.py`, `bot/handlers/start/ingest.py`, `bot/utils.py`, `backend/app/worker/processing.py`, `backend/app/api/users.py`

## Назначение и границы

Покрывает: как бот реагирует на сохранение в тихом режиме (реакции вместо
текста), команду переключения `/silent`, дефолт нового юзера, что бот шлёт
текстом несмотря на тишину.

НЕ покрывает: сам пайплайн AI-обработки и классификации ([БТ-2](бт-02-ai-обработка.md)),
рендер списков задач ([БТ-4](бт-04-списки-задач.md)), флоу напоминаний
([БТ-5](бт-05-напоминания.md)). Здесь — только разница «тихо / текстом».

## Пользовательские истории

Как пользователь:
1. Я хочу кидать в бота заметки как в «Избранное» — без диалога, без обязанности реагировать на ответ.
2. Я хочу видеть, что бот принял (👀) и обработал (👍) сообщение, не читая текст.
3. Я хочу вернуть старое поведение с текстовыми подтверждениями, если тихий режим мне не зашёл.
4. Я хочу, чтобы важное — ошибки, офферы напоминаний, списки — бот всё равно показал явно, а не молча проглотил.

## Проблема / зачем так

Раньше на каждое сообщение бот отвечал двумя текстами: «⏳ Сохранено!
Обрабатываю...» и затем карточкой «✅ Заголовок, категория, теги». Кинул три
ссылки подряд — шесть сообщений в ответ. Это превращало «свалку мыслей» в
переписку: чат заполнялся ответами бота, каждое сообщение будто требовало
прочтения. Ощущение Telegram Saved Messages («кинул и забыл») терялось.

Сейчас по умолчанию бот молчит. На входящее сообщение он ставит реакцию 👀
(принял), а после обработки меняет её на 👍 (готово). Ноль сообщений в чате —
только эмодзи на исходном сообщении пользователя. Старое «болтливое» поведение
осталось как opt-in (`/silent off`).

Важная граница реализации: тихий режим глушит **только подтверждения
сохранения обычной закладки**. Офферы напоминаний, 3-кнопочный выбор, списки
задач, ошибки и dedup-вопросы шлются текстом в обоих режимах — иначе фичи стали
бы невидимыми. Это сознательное решение (см. комментарий в `reminder_offer.py`),
а не недоделка.

## Поведение (правила)

### Что подавляется / что остаётся в silent

| Событие | Silent (дефолт) | Verbose (`/silent off`) |
|---|---|---|
| Текст / форвард принят | 👀 реакция на сообщение | «⏳ Сохранено! Обрабатываю...» |
| Документ (PDF и т.п.) принят | 📄 реакция | «⏳ Сохранено! Обрабатываю...» |
| Обработка завершена (обычная закладка) | 👀 → 👍 | карточка «✅ Заголовок / категория / summary» + кнопки |
| Ошибка обработки | 👀 → 👎 + текст «⚠️ Не удалось обработать. Попробуй ещё раз или /help» (остаётся в чате) | «❌ Не удалось обработать» + детали |
| Короткое сообщение (< 15 символов, без URL) | текстовый вопрос «Сохранить "…" как заметку?» с кнопками (в **обоих** режимах) | то же |
| Список задач распознан | текстовый оффер «Сделать список?» (текст) | оффер с inline-кнопками |
| Reminder-intent | оффер «🔔 Создать напоминание?» / 3-кнопки / авто-создание — **текстом** | то же |
| Near-duplicate найден | текстовый вопрос «уже есть похожая, что делать?» (снимает 👀) | то же |
| Slash-команда (`/list`, `/search`, `/reminders` …) | текстовый ответ | текстовый ответ |
| Reply на сообщение бота (NL-правка, ответ на оффер) | текстовый ответ | текстовый ответ |
| Onboarding-подсказка (первое сохранение) | текст «💡 Совет: …» один раз | то же |

**Главное правило тишины:** в silent глушится только успешное
подтверждение **обычной** закладки. Всё, что требует решения пользователя
(офферы, выбор, дубли) или сигнализирует о проблеме (ошибки), показывается
текстом в любом режиме.

### Команда `/silent`

| Ввод | Результат |
|---|---|
| `/silent` | переключить (toggle): silent → verbose или наоборот |
| `/silent on` | включить тихий режим |
| `/silent off` | включить обычный (verbose) режим |

Ответ-подтверждение ephemeral — само удаляется через 8 секунд:
- включили: `🔕 Тихий режим включён. Буду подтверждать реакциями 👀→👍`
- выключили: `🔔 Обычный режим включён. Буду отвечать текстом.`

### Хранение и дефолт

Настройка — в `User.settings` (JSONB), ключ `silent_mode`. **Дефолт — тихий
режим (`true`)**: новый юзер, у которого ключ не выставлен, по умолчанию в
silent. Если бэкенд недоступен и настройку не прочитать — бот тоже считает
режим тихим (fail-safe в сторону тишины).

Бот кэширует значение `silent_mode` на 5 минут (по telegram_id), чтобы не
дёргать API на каждое сообщение. После `/silent` кэш сбрасывается сразу —
переключение действует с первого следующего сообщения.

### Как бот понимает «принял» в silent

Реакции best-effort: если Telegram не дал поставить реакцию (старый клиент,
тип чата), бот не падает — просто реакции не будет. Реакция остаётся на
**исходном сообщении пользователя**, бот её не снимает после 👍 — это маркер,
что бот обработал. Реакции не считаются сообщениями, поэтому `/clean` их не
трогает и они не засоряют чат.

## Use Cases

### UC-8.1. Сохранение в тихом режиме (успех)

- **Предусловия:** silent_mode = true (дефолт).
- **Триггер:** обычное текстовое сообщение ≥ 15 символов («идея для продукта на выходные»).
- **Основной сценарий:**
  1. Бот мгновенно ставит 👀 на сообщение.
  2. Закладка уходит в обработку (worker, `silent=True`).
  3. По завершении бот меняет реакцию на 👍. Ни одного текстового сообщения.
- **Альтернативные сценарии:**
  - A1. Обработка упала → 👎 + текст «⚠️ Не удалось обработать. Попробуй ещё раз или /help» (остаётся в чате, не ephemeral).
  - A2. Форвард / медиа с подписью → то же поведение (👀 → 👍).
  - A3. Документ (PDF) → реакция 📄 вместо 👀, далее так же.
- **Результат:** закладка сохранена, в чате только реакция.

### UC-8.2. Сохранение в обычном режиме (verbose)

- **Предусловия:** silent_mode = false (`/silent off`).
- **Триггер:** обычное текстовое сообщение.
- **Основной сценарий:**
  1. Бот отвечает «⏳ Сохранено! Обрабатываю...».
  2. После обработки редактирует это же сообщение в карточку: «✅ <заголовок>», категория, summary, кнопки «📖 Открыть / 🗑 Удалить / 📋 Все закладки».
- **Альтернативные сценарии:**
  - A1. Ошибка → «❌ Не удалось обработать» с деталями (постоянное сообщение).
- **Результат:** карточка-результат текстом, как до фичи.

### UC-8.3. Переключение режима командой `/silent`

- **Триггер:** `/silent` (или `on` / `off`).
- **Основной сценарий:**
  1. Бот читает текущее `silent_mode`, вычисляет новое (toggle или явный аргумент).
  2. PATCH `User.settings`, сброс локального кэша.
  3. Ephemeral-подтверждение, авто-удаление через 8с.
- **Альтернативные сценарии:**
  - A1. PATCH настроек упал → «Ошибка при обновлении настроек.», режим не меняется.
- **Результат:** следующее сообщение обрабатывается уже в новом режиме.

### UC-8.4. Reminder-оффер не глушится тишиной

- **Предусловия:** silent_mode = true. AI распознал reminder-intent в сообщении.
- **Триггер:** «надо позвонить в банк» (weak/decision-intent).
- **Основной сценарий:**
  1. Бот ставит 👀 → 👍 (закладка сохранена молча).
  2. **Отдельным текстовым сообщением** показывает оффер: «🔔 Создать напоминание?» + «✕», подсказка про reply со временем.
- **Альтернативные сценарии:**
  - A1. Decision-форма `single_reminder` с распознанным временем → reminder создаётся сразу, текст «🔔 Напомню …» (тоже текстом, не реакцией).
  - A2. Форма `needs_button_choice` → текстовое сообщение с 3 кнопками 📋/🔔/✕.
- **Правила:** silent НЕ подавляет ни оффер, ни авто-создание, ни выбор. Тишина — только для подтверждения обычной закладки.

### UC-8.5. Список задач в тихом режиме

- **Предусловия:** silent_mode = true. AI распознал task_list.
- **Триггер:** «купить молоко, хлеб, сыр».
- **Основной сценарий:**
  1. Бот ставит 👀.
  2. Worker шлёт текстовый оффер «📋 Похоже на список — N пунктов. Сделать закреплённый список?» с кнопками ✅/✕.
  3. «Да» → список рендерится (в silent — текст + reply-инструкция, без inline-кнопок-чекбоксов), закрепляется; исходное текстовое сообщение удаляется как дубль.
- **Альтернативные сценарии:**
  - A1. Источник — голос/медиа → исходное сообщение НЕ удаляется (это запись, а не дубль текста).
- **Результат:** закреплённый список; полная тишина к спискам не применяется (это исключение по дизайну).

### UC-8.6. Короткое сообщение — подтверждение в обоих режимах

- **Триггер:** короткий текст (< 15 символов, без ссылки), например «купить».
- **Основной сценарий:**
  1. Бот в любом режиме спрашивает текстом «Сохранить "купить" как заметку?» с кнопками ✅/❌.
  2. «Да» в silent → вопрос удаляется, на оригинал ставится 👍, закладка сохраняется молча (`silent=True`).
  3. «Да» в verbose → «⏳ Сохранено! Обрабатываю...» и далее карточка.
- **Правила:** запрос подтверждения коротких сообщений — текстом всегда, тишина на него не распространяется (иначе непонятно, сохранилось ли).

## Mini App

Затрагивает **частично — как заглушка**. На экране «Я» (`MeScreen`) в группе
«приватность» есть строка **«тихий режим»** со значением **«в боте»**. Это
статическая надпись: тап по ней вызывает тост **«в разработке»**
(`comingSoon`), переключения нет. Менять тихий режим из Mini App **нельзя** —
только командой `/silent` в боте. Строка существует, чтобы обозначить, что
настройка живёт, но управляется ботом.
