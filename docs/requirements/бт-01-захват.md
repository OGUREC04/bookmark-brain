# БТ-1: Захват и сохранение

> Бизнес-требования · living-doc «как ведёт себя сейчас»
> Обновлено: 2026-05-24
> Связано: ADR [`0002`](../decisions/0002-yandex-speechkit.md) (голос→текст) · код `bot/handlers/start/ingest.py`, `bot/handlers/media.py`, `bot/handlers/documents.py`, `backend/app/api/bookmarks.py`, `backend/app/services/article_fetcher.py`, `backend/app/worker/processing.py`

## Назначение и границы

Покрывает: приём сообщения любого типа (текст, ссылка, пересланное, фото/видео,
документ, голос/аудио/видео-кружок), первичную реакцию бота, первичное сохранение
закладки и постановку её в очередь AI-обработки. Плюс добавление мысли из Mini App.

НЕ покрывает: что AI делает с сохранённым (заголовок, теги, категория, саммари,
распознавание задач/напоминаний) — это [БТ-2](бт-02-ai-обработка.md). Дедупликация
после обработки — [БТ-3](бт-03-дедупликация.md). Списки задач — [БТ-4](бт-04-списки-задач.md).
Напоминания (включая голосовое «напомни …») — [БТ-5](бт-05-напоминания.md). Поиск
голосом/текстом — [БТ-7](бт-07-поиск-просмотр.md). Тихий режим — [БТ-8](бт-08-тихий-режим.md).

## Пользовательские истории

Как пользователь:
1. Я хочу скинуть ссылку в один тап и забыть — бот сам выкачает статью и разберёт.
2. Я хочу переслать боту пост из другого канала, и чтобы он сохранился целиком.
3. Я хочу надиктовать мысль голосом, когда печатать неудобно, и получить её текстом.
4. Я хочу кинуть PDF или документ и чтобы бот вытащил из него текст.
5. Я хочу понимать, что бот меня услышал, — реакцию или ответ сразу, не через минуту.
6. Я хочу добавить мысль прямо из Mini App, не переключаясь в чат с ботом.

## Проблема / зачем так

Закладка начинается с «скинул и забыл». Если на этом шаге бот молчит десять секунд
или роняет сообщение — доверие к нему рушится сразу, до всякого AI. Поэтому захват
разведён на два такта: бот **подтверждает приём мгновенно** (реакция или «⏳ Сохранено!
Обрабатываю...»), сохраняет сырьё в базу, и только потом фоновый воркер занимается
тяжёлым — выкачкой статьи, классификацией, эмбеддингами.

Отсюда же правило: текст, который дорого добыть, бот показывает в чате **до** записи в
базу. Транскрипция голоса и текст из PDF появляются в ответе сразу — если backend потом
упадёт, пользователь не теряет распознанное и может скопировать руками. Сообщение
`«Не удалось сохранить как закладку. Текст выше — можешь скопировать»` — про это.

Отдельная боль — медиа без подписи. Фото без caption бот сохранить не может (нечего
классифицировать), поэтому он не глотает его молча, а просит подпись. Для альбома
(Telegram шлёт каждое фото отдельным апдейтом с общим `media_group_id`) предупреждение
показывается один раз, а не на каждый кадр.

## Поведение (правила)

### Типы входа → что извлекается → реакция

| Тип входа | Что извлекается | Первичная реакция бота |
|-----------|-----------------|------------------------|
| Текст ≥ 15 символов | сам текст; URL из entities/regex (первый) | verbose: `⏳ Сохранено! Обрабатываю...`; silent: реакция 👀 |
| Текст < 15 символов без URL | текст | спрашивает «Сохранить "…" как заметку?» с кнопками ✅/❌ |
| Ссылка (URL в тексте) | URL → воркер выкачивает статью (title + текст) | как текст; статья тянется фоном, не на приёме |
| Пересланное (`forward_date`) | текст/caption + URL | как текст; источник = `bot_forward` |
| Фото / видео / стикер | **только caption** (подпись) | без подписи — просит добавить caption; с подписью — как текст |
| Документ PDF/DOCX/TXT/MD | извлечённый текст (+ caption), число страниц | реакция 📄 (или «Читаю документ...»), затем превью первых 500 символов |
| Голос / видео-кружок / аудио-файл | транскрипция (STT) | реакция 👂 (или «Распознаю...»), затем reply с распознанным текстом |

### Ссылка → выкачка статьи

URL фетчится **не на приёме, а в фоновом воркере** (первый шаг AI-обработки). Через
trafilatura извлекаются `title`, очищенный основной текст и язык. Текст обрезается до
20 000 символов. Таймаут запроса — 15 секунд, редиректы разрешены.

- Если статья выкачана и raw_text был коротким (< 200 символов — «просто ссылка») →
  для AI используется текст статьи. Иначе текст статьи **дописывается** к сообщению.
- Если выкачка не удалась (сеть, не-200, paywall, парс-ошибка) → `fetch_article` **никогда
  не бросает**, возвращает пустой результат, обработка идёт по голому raw_text. Закладка
  сохраняется в любом случае.

### Голос → текст (STT)

Голосовое, видео-кружок и аудио-файл скачиваются и транскрибируются (Whisper или
Yandex SpeechKit — см. [ADR 0002](../decisions/0002-yandex-speechkit.md)). Распознанный
текст бот **всегда показывает reply'ем** (прозрачность STT), затем сохраняет закладку с
тегом `#voice` и сохранённой транскрипцией.

После транскрипции текст маршрутизируется по интенту:
- «напомни …» → напоминание ([БТ-5](бт-05-напоминания.md));
- поисковая форма → поиск ([БТ-7](бт-07-поиск-просмотр.md));
- список задач → task_list-закладка ([БТ-4](бт-04-списки-задач.md));
- иначе → обычная закладка-заметка.

Лимиты STT: запись короче **2 секунд** отвергается (Whisper галлюцинирует на тишине).
Файл больше **20 МБ** не скачивается (лимит Telegram Bot API). Голос длиннее **30 секунд**
на Yandex без Object Storage отвергается с подсказкой настроить `YANDEX_S3_*`. Если STT
не сконфигурирован вовсе — «Голосовые сообщения пока не поддерживаются».

### Документы

Поддержаны **PDF, DOCX, TXT, MD** (по mime-type, с fallback по расширению). Прочие
форматы фильтр не пропускает — для них срабатывает обычная медиа-ветка (просьба о тексте).
Извлечённый текст обрезается до **50 000 символов** (на превью — «текст обрезан до 50k»).
Зашифрованный PDF и пустой документ отклоняются с понятным сообщением. Лимит размера — 20 МБ.

### Источник и идемпотентность

`source` фиксирует канал: `bot_message` (текст), `bot_forward` (пересланное / медиа),
`telegram` (голос/документ), `miniapp` (из Mini App). Повторная отправка того же
Telegram-сообщения (тот же `source` + `source_message_id`) **не плодит дубль** — backend
ловит unique-конфликт и возвращает уже существующую закладку (идемпотентность POST).
Это не то же, что смысловая дедупликация похожих закладок — она в [БТ-3](бт-03-дедупликация.md).

### После сохранения

Сразу после создания закладка ставится в очередь arq на `process_bookmark_task`. Всё
дальнейшее (выкачка статьи, классификация, теги, оффер напоминания) — фоновый воркер,
вне рамок захвата.

### Mini App

Mini App **умеет создавать** закладку — через FAB «+» → шит «новая мысль»: поле текста →
«сохранить» → `POST /bookmarks/` с `source=miniapp`, дальше тот же воркер. Кнопки
«вложение» (скрепка) и «голос» (микрофон) в шите **задизейблены** — подпись «вложения — в
боте». То есть Mini App покрывает только **текстовый** захват; ссылку можно вставить
текстом (бот разберёт). Голос, документы, фото, пересланные — **только через бот**.

## Use Cases

### UC-1.1. Текстовая мысль / ссылка

- **Триггер:** обычное текстовое сообщение (не команда, не reply на бота).
- **Основной сценарий:**
  1. Бот извлекает текст и URL (если есть).
  2. verbose: отвечает `⏳ Сохранено! Обрабатываю...`; silent: ставит 👀.
  3. Создаёт закладку (`source=bot_message`), ставит в очередь обработки.
- **Альтернативные сценарии:**
  - A1. Текст короче 15 символов без URL → «Сохранить "…" как заметку?» (✅/❌); сохраняет только при ✅.
  - A2. Сообщение — explicit-команда «напомни …» → уходит в reminder-флоу, закладка НЕ создаётся ([БТ-5](бт-05-напоминания.md)).
  - A3. Это reply на сообщение бота → обрабатывают reminders/tasks upstream, сюда не доходит.
  - A4. backend упал на создании → verbose: «Ошибка при сохранении»; silent: 👎 + эфемерная ошибка.

### UC-1.2. Пересланное сообщение

- **Триггер:** сообщение с `forward_date`.
- **Основной сценарий:**
  1. Бот извлекает текст/caption + URL.
  2. Сохраняет (`source=bot_forward`), реакция/статус как в UC-1.1.
- **Альтернативные сценарии:**
  - A1. Пересланное без текста (одиночное) → «Не могу извлечь текст из этого сообщения».
  - A2. Пересланный альбом без подписи → один раз на альбом: «Получил альбом, но без подписи. Добавь caption».

### UC-1.3. Фото / видео / стикер

- **Триггер:** `F.photo | F.video | F.sticker`.
- **Основной сценарий:**
  1. Есть caption → текст = caption, сохраняется (`source=bot_forward`).
  2. Реакция/статус как в UC-1.1.
- **Альтернативные сценарии:**
  - A1. Без caption (одиночное) → «Получил медиа, но без текста. Добавь подпись».
  - A2. Без caption в альбоме → предупреждение один раз на `media_group_id`.

### UC-1.4. Голосовое / видео-кружок / аудио

- **Триггер:** `F.voice | F.video_note | F.audio`.
- **Основной сценарий:**
  1. Бот ставит 👂 (или «Распознаю...»), скачивает файл, транскрибирует.
  2. Reply'ем показывает распознанный текст (с таймкодами, если запись длинная).
  3. Сохраняет закладку с тегом `#voice` + транскрипцией, ставит в очередь.
- **Альтернативные сценарии:**
  - A1. Запись < 2 сек → «Слишком короткое сообщение (менее 2 секунд)».
  - A2. Файл > 20 МБ → «Файл слишком большой … до 20 МБ».
  - A3. Голос > 30 сек на Yandex sync → подсказка настроить Object Storage.
  - A4. STT не настроен → «Голосовые пока не поддерживаются».
  - A5. STT упал → 👎 + «Ошибка распознавания».
  - A6. backend упал ПОСЛЕ транскрипции → «Не удалось сохранить … Текст выше — скопируй» (текст не теряется).
  - A7. Текст — «напомни …» / поиск / список → роутится в БТ-5 / БТ-7 / БТ-4.

### UC-1.5. Документ (PDF/DOCX/TXT/MD)

- **Триггер:** `F.document` с поддержанным форматом.
- **Основной сценарий:**
  1. Бот ставит 📄, скачивает, извлекает текст.
  2. Reply'ем показывает превью (первые 500 символов + число страниц).
  3. Сохраняет (`content_type=document`, `title`=имя файла), ставит в очередь.
- **Альтернативные сценарии:**
  - A1. Неподдерживаемый формат → проходит в медиа-ветку (просьба о тексте), как фото.
  - A2. Зашифрованный PDF → «PDF защищён паролем».
  - A3. Пустой документ → «… пустой — не нашёл текста».
  - A4. Текст > 50k символов → обрезается, в превью «текст обрезан до 50k».
  - A5. Файл > 20 МБ → «Файл слишком большой».
  - A6. backend упал ПОСЛЕ извлечения → «… Текст выше — скопируй».

### UC-1.6. Добавление мысли из Mini App

- **Триггер:** FAB «+» → шит «новая мысль» → ввод текста → «сохранить».
- **Основной сценарий:**
  1. `POST /bookmarks/` с `source=miniapp`, `raw_text`=текст.
  2. Воркер обрабатывает так же, как захват из бота.
  3. Список в Mini App перезагружается.
- **Альтернативные сценарии:**
  - A1. Пустой текст → кнопка «сохранить» неактивна.
  - A2. Вложение/голос → кнопки задизейблены, подпись «вложения — в боте».

## Корнер-кейсы (сводка)

| # | Кейс | Ожидаемое поведение |
|---|------|---------------------|
| 1 | Битая / недоступная ссылка | закладка сохраняется по raw_text; статья не тянется (fetch не бросает) |
| 2 | Ссылка без статьи (paywall / 404 / не-200) | то же — обработка по голому тексту |
| 3 | Голос-тишина (< 2 сек) | отклоняется, «Слишком короткое сообщение» |
| 4 | Неподдерживаемый формат файла | идёт в медиа-ветку → просьба добавить текст |
| 5 | Зашифрованный PDF | «PDF защищён паролем» |
| 6 | Пустой документ | «пустой — не нашёл текста» |
| 7 | Очень длинная статья | текст обрезается до 20 000 символов |
| 8 | Очень длинный документ | обрезается до 50 000 символов, помечается «обрезан» |
| 9 | Файл > 20 МБ (голос/документ) | не скачивается, «Файл слишком большой» |
| 10 | Пустое / без текста сообщение | «Не могу извлечь текст» / просьба о подписи |
| 11 | Короткий текст (< 15 симв) без URL | спрашивает подтверждение перед сохранением |
| 12 | Фото/видео без caption | просьба добавить подпись; для альбома — раз на `media_group_id` |
| 13 | Повторная отправка того же Telegram-сообщения | дубль не создаётся, возвращается существующая закладка |
| 14 | backend упал после транскрипции / извлечения | распознанный текст уже в чате, «Текст выше — скопируй» |
| 15 | STT / AI недоступен | голос: явная ошибка; текст/ссылка: сохраняется, AI отрабатывает позже фоном |
| 16 | Смысловой дубль похожей закладки | НЕ здесь — ловится после обработки ([БТ-3](бт-03-дедупликация.md)) |
